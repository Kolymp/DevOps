- git init – инициализирует новый репозиторий Git в текущей папке.
- git clone <url>** – клонирует удалённый репозиторий на локальную машину.
- **git add <файл>** – добавляет файл в индекс (staging area) для следующего коммита.
- **git commit -m "сообщение"** – создаёт коммит с указанным комментарием.
- **git status** – показывает состояние рабочей директории и индекса.
- **git log** – выводит историю коммитов.
- **git diff** – показывает незакоммиченные изменения в рабочей директории.
- **git diff --staged** – показывает изменения, уже добавленные в индекс.
- **git branch** – список локальных веток.
- **git branch <имя>** – создаёт новую ветку.
- **git checkout <ветка>** – переключается на указанную ветку.
- **git checkout -b <ветка>** – создаёт и сразу переключается на новую ветку.
- **git merge <ветка>** – вливает изменения из указанной ветки в текущую.
- **git rebase <ветка>** – переносит коммиты текущей ветки на конец другой ветки.
- **git reset <файл>** – убирает файл из индекса, сохраняя изменения в рабочей области.
- **git reset --soft <коммит>** – перемещает HEAD на указанный коммит, оставляя изменения в индексе.
- **git reset --hard <коммит>** – сбрасывает индекс и рабочую директорию до состояния указанного коммита.
- **git revert <коммит>** – создаёт новый коммит, отменяющий изменения указанного коммита.
- **git remote -v** – показывает список удалённых репозиториев и их URL.
- **git remote add <имя> <url>** – добавляет новый удалённый репозиторий. 
- **git fetch <удалённый>** – загружает изменения из удалённого репозитория, но не сливает их.
- **git pull <удалённый> <ветка>** – загружает изменения и выполняет слияние с текущей веткой.
- **git push <удалённый> <ветка>** – отправляет локальные коммиты в удалённый репозиторий.
- **git push origin --delete <ветка>** – удаляет ветку в удалённом репозитории.
- **git tag** – выводит список тегов.
- **git tag <имя>** – создаёт легковесный тег на текущем коммите.
- **git tag -a <имя> -m "сообщение"** – создаёт аннотированный тег с сообщением.
- **git push --tags** – отправляет все локальные теги в удалённый репозиторий.
- **git stash** – временно сохраняет незакоммиченные изменения и очищает рабочую директорию.
- **git stash list** – показывает список сохранённых изменений (stash).
- **git stash apply** – применяет последние сохранённые изменения, не удаляя их из stash.
- **git stash pop** – применяет последние изменения и удаляет их из stash.
- **git stash drop** – удаляет последние сохранённые изменения из stash.
- **git clean -n** – показывает, какие неотслеживаемые файлы будут удалены.
- **git clean -f** – удаляет неотслеживаемые файлы из рабочей директории.
- **git cherry-pick <коммит>** – применяет изменения указанного коммита в текущую ветку.
- **git bisect start** – начинает бинарный поиск (бисекцию) для поиска проблемного коммита.
- **git bisect bad** – помечает текущий коммит как содержащий ошибку.
- **git bisect good <коммит>** – помечает коммит как рабочий (без ошибки).
- **git blame <файл>** – показывает, кто и когда изменял каждую строку файла.
- **git grep "шаблон"** – ищет заданный текст во всех файлах репозитория.
- **git log --oneline --graph** – выводит историю коммитов в компактном виде с графом ветвлений.
- **git shortlog** – группирует коммиты по авторам и выводит статистику.
- **git show <коммит>** – показывает информацию о коммите (метаданные и изменения).
- **git mv <старое> <новое>** – перемещает или переименовывает файл и сразу добавляет в индекс.
- **git rm <файл>** – удаляет файл из рабочей директории и добавляет это изменение в индекс.
- **git archive** – создаёт архив (tar/zip) с файлами из указанного коммита.
- **git submodule add <url>** – добавляет подмодуль в текущий репозиторий.
- **git gc** – запускает сборку мусора и оптимизацию локального репозитория.
- **git fsck** – проверяет целостность базы данных Git и ищет повреждённые объекты.
