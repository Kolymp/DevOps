### Работа с контейнерами (запуск, остановка, удаление)
1. **`docker run <образ>`** — Создает и запускает контейнер из образа.
2. **`docker run -d <образ>`** — Запускает контейнер в фоновом режиме (detach).
3. **`docker run -it <образ> /bin/bash`** — Запускает контейнер в интерактивном режиме с доступом к bash.
4. **`docker run --name <имя> <образ>`** — Запускает контейнер с заданным именем.
5. **`docker ps`** — Показывает список **запущенных** контейнеров.
6. **`docker ps -a`** — Показывает список **всех** контейнеров (включая остановленные).
7. **`docker start <контейнер>`** — Запускает остановленный контейнер.
8. **`docker stop <контейнер>`** — Останавливает работающий контейнер (мягко).
9. **`docker kill <контейнер>`** — Принудительно останавливает контейнер (жестко).
10. **`docker restart <контейнер>`** — Перезапускает контейнер.
11. **`docker rm <контейнер>`** — Удаляет остановленный контейнер.
12. **`docker rm -f <контейнер>`** — Принудительно удаляет работающий контейнер.
13. **`docker container prune`** — Удаляет все остановленные контейнеры.

### Управление образами (Images)
14. **`docker images`** — Показывает список скачанных образов.
15. **`docker pull <образ>`** — Скачивает образ из реестра (например, Docker Hub).
16. **`docker rmi <образ>`** — Удаляет образ.
17. **`docker build -t <имя:тег> <путь>`** — Собирает образ из Dockerfile (тег — обычно версия).
18. **`docker image prune`** — Удаляет неиспользуемые (висячие) образы.

### Информация и логи
19. **`docker logs <контейнер>`** — Показывает логи (вывод) контейнера.
20. **`docker logs -f <контейнер>`** — Показывает логи в реальном времени (как `tail -f`).
21. **`docker inspect <объект>`** — Показывает подробную информацию в формате JSON о контейнере или образе.
22. **`docker top <контейнер>`** — Показывает запущенные процессы внутри контейнера.
23. **`docker stats`** — Показывает статистику использования ресурсов (CPU, RAM) по всем контейнерам.
24. **`docker version`** — Показывает версию Docker.
25. **`docker info`** — Показывает информацию о системе Docker.

### Сеть (Networks)
26. **`docker network ls`** — Список сетей.
27. **`docker network create <имя>`** — Создает новую сеть.
28. **`docker network connect <сеть> <контейнер>`** — Подключает контейнер к сети.
29. **`docker network disconnect <сеть> <контейнер>`** — Отключает контейнер от сети.
30. **`docker network inspect <сеть>`** — Просмотр настроек сети.
31. **`docker network prune`** — Удаляет все неиспользуемые сети.

### Тома и данные (Volumes)
32. **`docker volume ls`** — Список томов.
33. **`docker volume create <имя>`** — Создает том (для хранения данных).
34. **`docker volume inspect <том>`** — Информация о томе.
35. **`docker volume rm <том>`** — Удаляет том.
36. **`docker volume prune`** — Удаляет все неиспользуемые тома.

### Взаимодействие с контейнерами
37. **`docker exec -it <контейнер> /bin/bash`** — Заходит внутрь **работающего** контейнера (открывает bash).
38. **`docker cp <источник> <контейнер>:<путь>`** — Копирует файл с хоста в контейнер.
39. **`docker cp <контейнер>:<путь> <назначение>`** — Копирует файл из контейнера на хост.
40. **`docker commit <контейнер> <новый_образ>`** — Создает новый образ из изменений текущего контейнера (используется редко, лучше Dockerfile).

### Публикация портов и переменные окружения
41. **`docker run -p 8080:80 <образ>`** — Пробрасывает порт: (порт хоста 8080) -> (порт контейнера 80).
42. **`docker run -e MY_VAR=value <образ>`** — Задает переменную окружения в контейнере.
43. **`docker port <контейнер>`** — Показывает проброс портов для контейнера.

### Очистка системы
44. **`docker system df`** — Показывает, сколько места занято образами, контейнерами и томами.
45. **`docker system prune`** — Удаляет всё неиспользуемое (контейнеры, сети, образы).
46. **`docker system prune -a --volumes`** — Тотальная зачистка (включая все неиспользуемые образы и тома).

### Docker Hub / Реестры
47. **`docker login`** — Вход в Docker Hub (или другой registry).
48. **`docker push <имя_пользователя>/<репозиторий>`** — Загружает образ в реестр (например, на Docker Hub).
49. **`docker search <термин>`** — Ищет образы на Docker Hub.

### Разное
50. **`docker history <образ>`** — Показывает историю слоев образа (как он был собран).
51. **`docker stop $(docker ps -q)`** — Останавливает **все** работающие контейнеры.  
    _(Команда `docker ps -q` возвращает только ID контейнеров, а `docker stop` останавливает их)._
52. **`docker rm $(docker ps -a -q -f status=exited)`** — Удаляет **все** контейнеры, которые уже завершили работу (Exited).  
    _(`-f status=exited` фильтрует контейнеры по статусу)._
53. **`docker rmi $(docker images -f "dangling=true" -q)`** — Удаляет все "висячие" образы (те, у которых нет тега, обычно `<none>`), чтобы освободить место.
54. **`docker system prune -a --volumes`** — Радикальная очистка системы. Удаляет **все** остановленные контейнеры, все неиспользуемые сети, все образы, на которые нет ссылок, и все неиспользуемые тома.  
    _(Будьте осторожны: удалит и volumes с данными, если они не используются контейнерами)._
55. **`docker run -it --rm -e DISPLAY=$DISPLAY -v /tmp/.X11-unix:/tmp/.X11-unix <образ_с_gui>`** — Запуск контейнера с графическим интерфейсом (Linux). Пробрасывает сокет X11, чтобы окно приложения отображалось на хосте.
56. **`docker logs -f --tail 100 <контейнер>`** — Показывает последние 100 строк лога контейнера и продолжает выводить новые сообщения в реальном времени.
57. **`docker run -d -v /host/data:/app/data --name app <образ>`** — Монтирует папку с хоста `/host/data` в папку контейнера `/app/data`. Если файл на хосте изменится, контейнер это увидит, и наоборот (bind mount).
58. **`docker run -d -p 8080:80 --restart unless-stopped --name web nginx`** — Запускает Nginx в фоне, пробрасывает порт и настраивает политику перезапуска: контейнер будет автоматически запускаться при ребуте докера или падении, но не запустится, если его остановили вручную.
59. **`docker exec <контейнер_бд> sh -c 'exec mysqldump -uuser -ppass db' > backup.sql`** — Создает дамп базы данных (MySQL), запуская команду `mysqldump` _внутри_ контейнера, и сохраняет результат в файл `backup.sql` на хосте.
60. **`docker cp <контейнер>:/var/log/app.log ./`** — Копирует файл логов из контейнера в текущую папку на хосте.
